Week 5 - Technical Blog

Ruby Classes

Like many other computer programming languages, Ruby uses classes as a way to organize methods. Ruby has many built in classes, like FixNum, String, and Array. Each of the built in classes in Ruby come with predefined methods that are used to manipulate data within those classes. As with the rest of Ruby, classes are objects and can be manipulated with methods as well.

Using classes when developing applications is useful because the methods that are defined within the class can only be used in that class. This helps applications become more modular in their design and less dependent on other code within the application. This is positive because if one part of the application needs to be changed, you’ll be able to rest easy in knowing that there’s not a ton of other methods or classes relying on that particular class and if changing it, the entire application won’t break.

It is often recommended that for well-designed code, classes should have a single purpose. When every method within a class contributes to the central purpose of that class, the code is considered highly cohesive. This is a good thing because it’s well organized and easy to read.

Below is an example of a class that we can use to help us calculate the perimeter and area of a square based on the side length.

class Square
def initialize(side_length)
@side_length = side_length
end

def side_length
@side_length
end

def perimeter
perimeter = (@side_length * 4)
end

def area
area = (@side_length * @side_length)
end
end

The first line in the above example is where the class is defined. In this case, I’ve named the class Square. Below the class name are four instance methods: initialize, side_length, perimeter, and area. These methods can only be used on an object of the Square class. The initialize method is used to create a new square object and side_length is the argument that is used to define the variable. For example, to create a new square, I would type:

my_own_square = Square.new(5)

Here I have a new square with the side length of 5.

If I wanted to learn the perimeter and area, I could call the two instance methods I defined above. For instance,

p my_own_square.perimeter # return >> 20
p my_own_square.area #return >> 25

When I call the perimeter and area method on my_own_square object, it returns the values of the my square’s perimeter and area.

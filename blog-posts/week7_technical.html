Week 7 - Technical Blog

SQL vs NoSQL

SQL refers to relational database models, which are databases that store information and also how it that information is related. Relational databases use schemas to organize information. Each schema is a table that has the ability to relate to other tables based on a certain ID.

Pros Cons
  • SQL databases take advantage of structured query language to access data, which has decades of support
  • High ability to manipulate data
  • Vertically Scalability (i.e. performance improvements come from upgrades in hardware)
  • Limited to only handling structured data
  • Little ability to scale across multiple servers
  • More management required compared to NoSQL

The focus of NoSQL databases is to provide scalability, performance, and high availability. Unlike of relational databases, NoSQL organizes information much more broadly, which can be accessed quicker, although not as specific.

Pros Cons
  • Can handle structured and unstructured data
  • More performance relative to SQL
  • Ability to take advantage of other query languages (other than SQL)
  • Horizontal Scalability (i.e. performance increases come from number of servers used to store data)
  • Less Functionality
  • No Joins Support
  • No Complex Transaction Support
  • No Constraints Support

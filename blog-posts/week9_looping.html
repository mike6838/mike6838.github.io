Week 9 - Technical Blog

Looping in JavaScript and Ruby

Both Ruby and JS have the ability to loop. The major difference between these two languages are the different ways we can go about looping. For instance, JS has only two ways of looping, where as ruby has many more. Below is an example of a for loop in JS.

var someArray = [1, 2, 3, 4, 5];

var someForLoop = function(n) {
sum = 0
for (i = 0; i < n.length; i++) {
sum += n[i]
};
console.log(sum);
};

someForLoop(someArray)

Above we have an array named someArray that we want to loop through and add all the values up. Within the array, we define a variable sum as 0, which we'll use to add all the elements of someArray. Underneath of where we defined the sum variable, we begin to write out for loop. I've defined i as zero, so that we can tell the function to continue as long as i is less than the length of the array we use as our argument, while everytime the loop is called, 1 will be added to i.

Below we have an example of a while loop in JS.

var someArray = [1, 2, 3, 4, 5];

var i = 0
var sum = 0
var someWhileLoop = function(n) {
while (n.length > i++) {
sum += n[i]
};
console.log(sum);
};

someWhileLoop(someArray)

Above, you'll notice that I've defined the variables i and sum outside of the while loop function. This is because in a while loop, everytime it's called to loop, it resets the variables. Consequently, if we defined those two variables inside the function like we did with the for loop, the loop would never stop because i would never get above 0.

Below is an example of a built in Ruby method that facilitate looping.

some_array = [1, 2, 3, 4, 5]

def add(n)
puts n.inject(:+)
end

add(some_array) => 15

The above example exemplifies some of the built in methods Ruby has that almost seem tailor made for certain jobs.
